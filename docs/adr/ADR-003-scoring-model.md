# ADR-003: 採点は重み付きルーブリックで多軸評価する

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

「どの候補を採用すべきか」を再現可能な形で決める必要がある。単一のテスト合否だけでは、性能・安全性・保守性の差が反映されない。

## 決定

5 軸の重み付きルーブリック（correctness 50 / security 20 / performance 15 / maintainability 10 / resource_usage 5、合計 100）で採点する。重みは `src/scoring.rs` の定数を単一の真実源とし、Phase 2 で `safecode.toml` から上書き可能にする。

## 理由

- 重みを明示することで採用判断の根拠が説明可能になる。
- 軸を独立させることで、各軸の実測を段階的に追加できる（MVP は correctness のみ実測、他は 0 点）。
- 重みを設定ファイル化すれば、用途（性能重視/安全重視）ごとに方針を切り替えられる。

## トレードオフ

- 重み配分は主観であり、ドメインによって最適値が変わる（チューニングが必要）。
- MVP では correctness 以外が 0 点のため、総合スコアは当面 0〜50 に偏る（Phase 3 まで暫定）。
