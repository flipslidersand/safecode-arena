# ADR-001: Phase 1〜4 はサブプロセス分離、Phase 5 で Wasmtime に移行

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

AI 生成コードの実行環境として、最初から Wasmtime サンドボックスを使うか、
まずサブプロセス分離で実装するかの選択がある。

## 決定

Phase 1〜4 は `tokio::process::Command` + `SIGKILL` タイムアウトのサブプロセス方式で実装し、
Phase 5 で Wasmtime + WASI に移行する。

## 理由

- サブプロセス方式は `cargo test` をそのまま実行できるため、Phase 1〜4 の評価ロジック実装に集中できる
- Wasmtime は「Rust → Wasm コンパイル + WASI 設定」の学習コストが高く、評価ロジックと同時に実装すると複雑さが爆発する
- OS レベルのプロセス分離で Phase 4 まではセキュリティ要件を満たせる（開発・ローカル環境を想定）

## トレードオフ

- Phase 4 まではファイルシステム・ネットワークへの完全制限ができない
- Phase 5 への移行時に `runner.rs` の実行パスを書き換える必要がある
