# ADR-003: スコアを重み付き加重平均で計算する

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

複数の評価軸（正確性・安全性・性能・保守性・リソース）を1つのスコアに集約する方法が必要。

## 決定

`spec.yaml` に重みを定義し、各軸のスコアを `0.0〜1.0` に正規化した上で加重平均を取る。
デフォルト重み: correctness 50 / security 20 / performance 15 / maintainability 10 / resource_usage 5。

## 理由

- 重みを YAML で外部化することで、プロジェクトごとに評価基準を変えられる
- 加重平均は解釈が直感的で、採点ロジックのデバッグが容易
- 将来的に重みをユーザーが CLI 引数で上書きできる拡張が容易

## トレードオフ

- 軸間の相関（例: 正確性が低ければ性能も意味がない）を無視している
  → correctness が 0 の場合は総合スコアを強制的に 0 にするルールで補完する
