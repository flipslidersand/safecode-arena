# ADR-004: 評価結果の永続化に SQLite を使う

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

評価結果を後から参照・比較するために永続化が必要。
JSON ファイル / SQLite / PostgreSQL の選択肢がある。

## 決定

`rusqlite` (bundled) で SQLite に保存する。

## 理由

- 評価ツールの性質上、サーバー型 DB を別途立ち上げる手間を避けたい
- `results.db` 1ファイルで完結するため、チームへの共有やバックアップが容易
- `compare` コマンドで複数候補を比較する際に SQL クエリが使える

## トレードオフ

- 並行書き込みが弱い（WAL モードで緩和可能）
  → 評価は候補ごとにシーケンシャル実行するためほぼ問題なし
