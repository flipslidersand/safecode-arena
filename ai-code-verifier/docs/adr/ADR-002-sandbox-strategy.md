# ADR-002: MVP のサンドボックスはプロセス隔離から始める

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

AI 生成コードは無限ループ、過剰なメモリ確保、ファイル/ネットワークアクセスなど危険な挙動をしうる。理想は WASI による capability ベースの完全隔離だが、Wasmtime 組み込みは実装コストが高い。

## 決定

MVP では「一時ディレクトリ + 子プロセス + タイムアウト」によるプロセス隔離を採用し、Wasmtime/WASI による強サンドボックスは Phase 4 へ後回しにする。

## 理由

- まず一気通貫のパイプライン（compile → test → 採点 → レポート）を最短で動かすことが最優先。
- プロセス隔離 + タイムアウトでも、無限ループや明白な暴走は止められ、MVP の価値は出せる。
- Wasm 化は対象コードのビルド方式に制約を課すため、設計が固まってから導入する方が手戻りが少ない。

## トレードオフ

- MVP の段階ではファイルシステム/ネットワークへのアクセスを完全には封じられない（同一ユーザ権限で動く）。信頼できない第三者コードを本番で回す用途には不十分。
- タイムアウトは Phase 1 では事後判定（`wait_timeout` 導入は Phase 2）であり、即時 kill ではない。
