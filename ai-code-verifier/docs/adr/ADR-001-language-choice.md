# ADR-001: 実装言語に Rust を採用する

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

検証ランナーは「信頼できない AI 生成コードを実行・制御する」ツールであり、ランナー自身が堅牢かつ高速である必要がある。プロセス制御、リソース制限、将来的な WebAssembly サンドボックス（Wasmtime）の組み込みが要件に含まれる。

## 決定

実装言語に Rust（2021 edition）を採用する。

## 理由

- 所有権モデルにより一時リソース（プロセス、一時ディレクトリ、ファイルハンドル）のリークを型レベルで防げる。
- `wasmtime` が Rust ネイティブで、Phase 4 のサンドボックス強化に直結する。
- Go も候補だったが、検証対象自体が Rust コードであり（MVP）、`cargo` 連携・Tree-sitter・cargo-fuzz と同一エコシステムで完結できる利点が大きい。
- 学習目的（所有権・エラー処理・プロセス制御）にも合致する。

## トレードオフ

- コンパイル時間と学習コストが Go より高い。
- 多言語対応（Python/Go/JS）を足す際、各言語ツールチェーンの外部呼び出しが必要になる（Rust だけでは閉じない）。
